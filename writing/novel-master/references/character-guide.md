# 角色创建与一致性维护指南

## 一、角色设计方法论

### 1. 核心三角
每个重要角色需定义三个彼此矛盾/张力的要素：
- **欲望**（Want）：角色认为自己想要的
- **需求**（Need）：角色真正需要成长/改变的
- **恐惧**（Fear）：阻碍角色的内在障碍

> 示例：修仙男主——欲望：变强复仇；需求：学会信任他人；恐惧：再次失去所爱

### 2. 性格钻石模型
四个维度建立角色辨识度：
- **态度维度**：对世界是乐观/悲观/务实/理想主义
- **行动维度**：主动型/被动型/反应型
- **社交维度**：外向/内敛/伪装型（表里不一更有层次）
- **道德维度**：守序/混乱/功利/原则

### 3. 角色区分度清单
同一作品中的角色必须在以下至少3个维度上彼此区分：
- 说话方式（句长、用词、语气词）
- 面对危机的第一反应
- 幽默风格（冷笑话/毒舌/自嘲/不会开玩笑）
- 价值排序（生命 vs 荣誉 vs 利益 vs 感情）
- 肢体语言习惯

## 二、对话区分度模板

### 语言特征设计
```
角色A（世家贵女）：
- 句式：多用反问、委婉否定
- 用词：书面语偏多，偶尔引经据典
- 语气词：极少，保持端庄
- 示例："此事……怕是不妥吧。"

角色B（市井少年）：
- 句式：短句为主，感叹句多
- 用词：口语化，夹带俚语
- 语气词："嘿""得了""切"
- 示例："嘿！这买卖，干了！"
```

## 三、人物弧光追踪

### 弧光类型
| 类型 | 描述 | 适用 |
|------|------|------|
| 正向弧光 | 角色克服缺陷，获得成长 | 大多数主角 |
| 负向弧光 | 角色逐步堕落或被环境吞噬 | 反派、悲剧主角 |
| 平坦弧光 | 角色本身不变，改变周围世界 | 信念型主角（如超人型） |
| 波动弧光 | 成长→回退→再成长 | 真实感最强，适合长篇 |

### 弧光检查点
每 10 章进行一次弧光审计：
1. 主角的核心信念是否有所动摇/强化？
2. 关键关系是否发生了质变（非量变）？
3. 角色的决策模式是否体现了成长？
4. 读者能否明确感知到"这个角色和开头不一样了"？

## 四、一致性校验规则

### 写作时自检
每次输出正文前，内部校验：
- ✅ 角色当前已知的信息范围（不能让角色知道他不应该知道的事）
- ✅ 角色的能力上限（不能突然超常发挥无解释）
- ✅ 角色的情感状态延续（上章刚伤心，这章不能无缘无故开心）
- ✅ 角色对其他角色的称呼是否统一
- ✅ 角色的行为是否符合已建立的性格（若需要反常行为，必须有充分铺垫或后续解释）

### 常见一致性Bug
| Bug | 示例 | 修复 |
|-----|------|------|
| 信息穿越 | 角色突然知道不该知道的秘密 | 补铺垫或删除 |
| 能力膨胀 | 实力忽然暴涨无交代 | 加修炼/机缘描写 |
| 性格漂移 | 冷酷角色突然话多热情 | 回溯加情感触发事件 |
| 关系跳跃 | 敌人突然变密友无过渡 | 加共患难/误会澄清桥段 |
| 称呼混乱 | 同一人物被不同称呼无规律 | 统一称呼表 |

## 五、群像管理

### 出场管理
- 主角：每章必须出场或被提及
- A级配角（3-5人）：每 3 章至少出场一次
- B级配角：有剧情需要时出场，需有提示唤起读者记忆
- 龙套：可复用但不要让读者误以为是重要角色

### 关系网络
维护一份隐式关系矩阵：
```
角色A → 角色B：表面[竞争对手] / 内心[暗恋]
角色A → 角色C：表面[师徒] / 内心[愧疚]
角色B → 角色C：表面[无关] / 伏笔[其实是失散的兄妹]
```
确保每对重要关系都有"表层关系"和"深层关系"两个维度。
